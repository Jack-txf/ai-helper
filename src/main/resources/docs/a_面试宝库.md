1.HashMap扩容时链表转红黑树的阈值为什么是8？退化为链表的阈值为6的原因？
红黑树单个节点的空间为链表的两倍，长度低时红黑树优势不大。

2.synchronized锁升级过程？
Synchronized的锁升级过程主要分为三步：
首先是偏向锁阶段。当一个线程第一次获得某个对象的锁时，JVM会在对象头中记录这个线程的ID，表示该锁“偏向”这个线程。这样做的目的是减少无竞争情况下的同步开销，因为后续这个线程再次进入同步块时，不需要进行任何同步操作，直接进入，避免了CAS操作和系统调用。偏向锁适合单线程访问的场景，极大提升了性能。
接下来，如果有第二个线程尝试竞争这个锁，偏向锁就会被撤销，锁会升级为轻量级锁。轻量级锁的实现依赖于CAS操作，线程会尝试通过CAS将对象头中的锁标记替换为自己的锁记录（Lock Record），如果成功，线程获得锁；如果失败，说明有竞争，锁会进一步升级。轻量级锁适合短时间的锁竞争，避免了重量级锁带来的系统调用开销。
最后，当竞争变得激烈，比如多个线程频繁争抢同一把锁，轻量级锁无法有效解决竞争问题时，锁会升级为重量级锁。重量级锁是基于操作系统的互斥量（Mutex）实现的，线程会进入阻塞和唤醒机制，保证线程安全，但代价是上下文切换和系统调用带来的性能损耗。
3.接口和抽象类的应用场景
抽象类：由于抽象类（abstract class）可以包含抽象方法（无实现）和具体方法（有实现），还能定义成员变量。其场景：
存在 "is-a" 关系的类层次结构，且有共享属性和行为：
抽象类可以定义带状态的成员变量（如private int count），子类可继承并修改这些状态。
例如：AbstractConnection抽象类封装数据库连接的共性（url、username等属性，connect()、disconnect()的默认实现），子类MySQLConnection、OracleConnection继承后仅需修改数据库特有的细节。
模板方法模式：
抽象类定义算法的骨架（固定流程），将可变步骤声明为抽象方法，由子类实现具体细节。
例如：
● 框架中的AbstractServlet，定义了处理 HTTP 请求的固定流程（service()方法），将doGet()、doPost()等可变步骤设为抽象方法，由具体 Servlet 子类实现。
● 报表生成工具的AbstractReportGenerator，封装 "读取数据→处理数据→生成报表" 的固定流程，将 "处理数据" 的具体逻辑留给子类实现。

接口：接口（interface）主要定义行为规范（Java 8 + 可包含默认方法和静态方法），一个类可实现多个接口。其场景：
多实现需求：Java 是单继承，接口可让类同时具备多种能力。例如：一个Student类，可同时实现Comparable（支持排序）、Serializable（支持序列化）、Cloneable（支持克隆）三个接口，具备三种不同能力。
定义跨类别的行为规范（"has-a" 能力）
当多个无关类需要具备相同的行为（但不属于同一继承体系），用接口统一行为标准。
例如：
● Runnable接口：定义run()方法，无论Thread的子类还是普通类，只要实现Runnable，就能被线程执行（无关类都可具备 "可运行" 能力）。
● Comparable接口：定义compareTo()方法，Integer、String、User等完全不同的类，实现后都可被排序（具备 "可比较" 能力）。

● 用抽象类当：
○ 需要定义父子类关系（如 Animal → Cat）
○ 共享字段或非公有方法
○ 提供部分实现的模板
● 用接口当：
○ 定义跨继承树的能力（如 Flyable 可被飞机、鸟类实现）
○ 需要多重行为组合（如 class Robot implements Walkable, Talkable）
○ 声明服务契约（如 DAO、Service 层接口）
💡 最佳实践：Java 8 后，优先用接口定义行为（通过默认方法提供基础实现），用抽象类处理深度继承关系中的状态共享。
● is-a 用于描述 "类型归属"，适合继承关系（如 "狗是动物"），侧重类的层级和共性复用。
● has-a 用于描述 "包含关系"，适合组合 / 聚合（如 "汽车有发动机"），侧重功能的灵活组合，是面向对象设计中 "优先使用组合而非继承" 原则的核心体现。

4.为什么可以String a = "abc"？
在 Java 中，String s = "abc" 这种直接赋值的创建方式，是 Java 对字符串（String）类型的特殊优化，核心原因是字符串常量池（String Constant Pool） 的存在，以及 Java 编译器和 JVM 对字符串的特殊处理。
当你使用 String s = "abc" 时，Java 的处理流程是：
● 首先检查字符串常量池中是否已经存在 "abc" 这个字符串对象。
● 如果存在，直接让引用变量 s 指向常量池中的这个对象（复用已有对象）。
● 如果不存在，在常量池中创建 "abc" 对象，再让 s 指向它。
